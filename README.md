# Թվայնացված Թանգարանային Նմուշների Կայք

Այս նախագիծը փոքր Flask հավելված է՝ նախատեսված թանգարանային նմուշների թվայնացման, պահպանման և ներկայացման համար։

Հիմնական հնարավորություններ
- Օգտագործողների գրանցում և մուտք
- Պրոֆիլ (պատկերի վերբեռնում, կարգավորումներ՝ օգտատիրոջ տվյալները փոխելու հնարավորություն)
- Նմուշների (Artifact) ստեղծում/խմբագրում/ջնջում՝ նկարչային ֆայլերի և թամբնեյլների հետ
- Ցուցադրություններ (Exhibitions)՝ նմուշների խմբավորումներ
- Հեշտ տեղական թեստավորման սցենարներ

Տեխնոլոգիաներ
- Python 3.8+
- Flask
- Flask-SQLAlchemy (SQLite)
- Flask-Login
- Pillow (PIL) — պատկերների մշակման համար

Տեղադրում և տեղական գործարկում (PowerShell)

1. Առաջին հերթին՝ (խորհուրդ) ստեղծեք և ակտիվացրեք վիրտուալ միջավայր
```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
```

2. Տեղադրեք ազդակառուները
```powershell
python -m pip install --upgrade pip
python -m pip install flask flask-sqlalchemy flask-login pillow
```

3. Թարմացրեք տվյալների բազան՝ ավելացնող սյուները (եթե դեռ չեք արել)
```powershell
python update_db.py
```

4. Գործարկեք development սերվերը
```powershell
python app.py
```

5. Բացեք բրաուզերը
```text
http://127.0.0.1:5000
```

Ֆայլեր և տեղաշարժ
- Static ֆայլեր՝ `static/` (լոգո՝ `static/logo.svg`, նախադիտման պատկերներ՝ `static/default-artifact.jpg`, `static/default-profile.png`)
- Վերբեռնված ֆայլերը պահվում են `static/uploads/`-ում
- Տվյալների բազա՝ `instance/museum.db` (SQLite)

Ավելացրած հնարավորություն (շեշտ)` (կարճ)
- Պրոֆիլային նկարների վերբեռնում և պահպանություն
- Կարգավորումների էջ՝ օգտատիրոջ տվյալների (username/email/password/profile image) փոփոխության հնարավորություն
- Անվտանգության բարելավում՝ գաղտնաբառ փոփոխելիս պահանջվում է ընթացիկ գաղտնաբառը

Թեստեր
- Տեղում կան պարզ թեստ-սքրիպտեր՝ `test_settings.py` և `test_user_settings_flow.py`։
	- Թեսթերը օգտագործում են Flask test client՝ local integration checks համար։

Թեստեր գործարկելու օրինակ
```powershell
python test_settings.py
python test_user_settings_flow.py
```

Նշումներ և առաջարկություններ
- Միգրացիաները ներկայումս իրականացվում են `update_db.py` միջոցով, որը ուղակի օգտագործում է `ALTER TABLE` հրամաններ։ Production-ի համար խորհուրդ է տրվում անցնել Alembic/Flask-Migrate օգտագործման՝ վերահսկվող միգրացիաների համար։
- Պատկերների չափերի և ֆորմատների ստուգումը կատարվում՝ թույլատրելի են `jpg, jpeg, png, gif, webp`։
- Եթե լոգոն չի երևում՝ համոզվեք, որ `static/logo.svg` գոյություն ունի և եթե պետք է՝ հղումը փոխել `base.html`-ում։

Եթե ուզում եք, ես կարող եմ ավելացնել՝
- OAuth մուտք (Google/Facebook)
- Ավտոմատացված test-suite (pytest + fixtures)
- Պրոֆիլային լուսանկարի նախադիտում առաջադրվելուց առաջ (client-side)
